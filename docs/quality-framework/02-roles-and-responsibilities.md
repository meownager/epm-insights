# Roles and Responsibilities

## Overview

This document defines who owns each part of the audit process — criteria, results, reports, and the quality framework itself.

## Role Definitions

### Audit System Owner

**Currently:** Syeda M. (smonowar@purdue.edu)

Responsibilities:
- Owns the Audit Criteria Register and approves all threshold changes
- Reviews audit results and determines follow-up actions
- Approves generated reports before sharing with external parties
- Maintains quality framework documents and initiates the calibration and review cycle
- Determines when corrective actions are complete

### Project Manager (per project)

**Currently:** R. Patel, S. Reyes, M. Chen, T. Nguyen, D. Okafor (as recorded in proposal data)

Responsibilities:
- Provides accurate proposal data at project kickoff
- Provides actual data at project close (budget, hours, dates, resource count, closeout notes)
- Reviews audit findings for their projects and raises disputes if data is incorrect
- Does not modify audit thresholds or health classifications

### Future Reviewer / External Auditor

Responsibilities:
- Reviews the audit framework and run records for process compliance
- Does not change criteria or results — review is read-only

## Criteria Ownership

The Audit Criteria Register (`config/audit_criteria.yaml`) is owned exclusively by the Audit System Owner. Changes require:
1. A version bump
2. A recorded reason in the `change_reason` field
3. A note in the Findings and Corrective Action Log if the change was triggered by a finding

No other person or system may modify thresholds.

## Report Approval

Reports generated by the system are advisory outputs. The Audit System Owner reviews them before sharing. AI-assisted narrative drafts (Phase 5) are always reviewed and approved by the Audit System Owner before use.

## Separation of Duties

| Action | Who May Perform It |
|---|---|
| Change audit thresholds | Audit System Owner only |
| Run an audit | Audit System Owner |
| Review findings | Audit System Owner + Project Manager (own projects) |
| Approve reports for sharing | Audit System Owner |
| Modify source code | Audit System Owner |
| Provide project data | Project Manager |
